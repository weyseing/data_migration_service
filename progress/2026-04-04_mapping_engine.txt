Step 2: Mapping Engine — COMPLETED

Uses SQLGlot for deterministic type mapping + Claude Haiku (cheapest model) for stored procedure conversion.

=== Step 2.1: Data Models ===
Created: backend/models/mapping.py
  - ColumnMapping: source_column, source_type, target_type, notes
  - TableMapping: source_table + list of ColumnMapping
  - ProcedureMapping: source_name, source_body, target_sql, explanation
  - MappingResult: combines table_mappings + procedure_mappings

=== Step 2.2: Type Mapper (SQLGlot) ===
Created: backend/mapping/type_mapper.py
  - map_column_type(col) — maps a single MySQL column type to PostgreSQL
  - map_table(table) — maps all columns in a table
  - map_all_tables(tables) — maps all tables
  - ENUM handling: ENUM('a','b') → TEXT CHECK (col IN ('a','b'))
  - UNSIGNED handling: strips UNSIGNED, adds note (PostgreSQL doesn't support it)
  - Uses SQLGlot transpile(read="mysql", write="postgres") for everything else
  - Examples: DATETIME → TIMESTAMP, VARCHAR stays VARCHAR, DECIMAL stays DECIMAL

=== Step 2.3: AI Mapper (Claude Haiku) ===
Created: backend/mapping/ai_mapper.py
  - Model: claude-haiku-4-5-20251001 (~$0.80/M input tokens, cheapest)
  - map_procedure(proc) — sends MySQL stored procedure to Claude, gets PostgreSQL PL/pgSQL back
  - Prompt asks Claude to: use PL/pgSQL, replace cursors, keep same logic, return only SQL
  - Strips markdown fences from response if present
  - Graceful fallback if ANTHROPIC_API_KEY not set (returns skip message)
  - map_all_procedures(procs) — maps all procedures

=== Step 2.4: Mapping Orchestrator ===
Modified: backend/mapping/__init__.py (was empty)
  - run_mapping(discovery) — calls type_mapper + ai_mapper, returns MappingResult

=== Step 2.5: API Endpoints ===
Modified: backend/main.py
  - Added _mapping_cache (module-level variable, same pattern as discovery)
  - POST /api/mapping/run — runs mapping on cached discovery result, returns MappingResult
  - GET /api/mapping/tables — returns table type mappings
  - GET /api/mapping/procedures — returns procedure mappings
  - Returns 400 if discovery hasn't been run yet

=== How to Use ===
  1. Open http://localhost:8000
  2. Click "Run Discovery" (scans MySQL)
  3. Click "Run Mapping" (maps types + converts procedures via AI)
  4. Expand any table to see MySQL → PostgreSQL type mappings
  5. Expand procedure to see side-by-side MySQL vs PostgreSQL SQL
